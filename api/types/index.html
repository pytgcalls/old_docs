Available Types¶

This page is about PyTgCalls Types. All types listed here are available through the pytgcalls.types package. Unless required as argument to a client method, most of the types don’t need to be manually instantiated because they are only returned by other methods. You also don’t need to import them, unless you want to type-hint your variables.

from pytgcalls.types import Update, AudioImagePiped, ErrorDuringJoin, ...

Note

Optional fields always exist inside the object, but they could be empty and contain the value of None. Empty fields aren’t shown when, for example, using print(update) and this means that hasattr(update, "participant") always returns True.

To tell whether a field is set or not, do a simple boolean check: if update.participant: ....
